# Research Digest AI

> An AI-native research assistant on **Anna** that turns any topic into a structured, reviewable knowledge digest — summary, key points, key concepts, and related ideas — then lets you approve and save it to a personal library.

Built for the **Anna AI-Native App Hackathon** (Jun 15–22, 2026).

**▶️ [Live demo](https://annaresearch.soenic.com) · 🎬 [Demo video](https://youtu.be/QdnF-ZqskO8)**

---

## TL;DR for Judges

- **What it is:** A focused research workflow, not a chatbot. You enter a topic → AI synthesizes a structured digest → **you review and approve** → it's saved to your library with persistent state.
- **Why it's AI-native:** The AI is the engine of every result. It produces *structured* output (typed JSON → UI cards), runs as an Anna **Tool**, and is guided by an Anna **Skill** for in-chat behavior.
- **How it fits Anna:** Uses `tools.invoke`, `storage`, and `chat.write_message`; ships a manifest (AppManifest v1), a Tool executa, and a Skill executa.
- **Try it in 60 seconds:** [Run locally](#run-it-locally-fastest-demo) — one command, no Anna account needed.

## The Workflow (one clear interaction)

```
   ┌─────────────┐    ┌──────────────────┐    ┌────────────────┐    ┌───────────────┐
   │ Enter topic │ →  │  AI synthesizes  │ →  │  Human review  │ →  │ Save / chain  │
   │ + pick depth│    │ structured cards │    │  approve/discard│   │ to library    │
   └─────────────┘    └──────────────────┘    └────────────────┘    └───────────────┘
```

1. **Enter a topic** — e.g. *"Byzantine fault tolerance"*, *"CRISPR gene editing"*, *"Stoic philosophy"*.
2. **Choose depth** — Quick (3 points), Standard (5), or Deep Dive (7).
3. **AI synthesizes** a digest into four structured cards:
   - **Summary** — 2–3 sentence executive overview
   - **Key Points** — the most important takeaways
   - **Key Concepts** — term + definition glossary
   - **Related Topics** — clickable suggestions to chain research
4. **Human review** — approve to **Save to Library**, or **Discard**. Nothing is stored until you say so.
5. **Library** — browse, reload, or delete saved digests in the sidebar; click a related topic to start the next digest instantly.

## How AI Is Used (meaningfully, not decoratively)

- Every digest is **generated by Claude** (`claude-opus-4-8`) inside the `research-processor` Tool.
- The model is constrained to return **strict, typed JSON** (title, summary, key_points, concepts, related_topics, confidence) which the UI renders as structured cards — this is *AI producing structured app state*, not free-form chat.
- A **confidence signal** (`high | medium | low`) is surfaced per digest so the human reviewer can judge trustworthiness.
- The `research-coach` **Skill** lets the same capability work conversationally inside Anna chat: it decides depth, calls the tool, and summarizes results with a save prompt — i.e. AI *participating in the workflow*, including the human-review handoff.

## How It Fits Anna

| Anna primitive | Where it's used |
|---|---|
| **App + Manifest (v1)** | [`manifest.json`](manifest.json) — slug, permissions, views, `host_api` |
| **Tool (executa)** | [`executas/research-processor-node/`](executas/research-processor-node/) — JSON-RPC 2.0 plugin over stdio (`research`, `get_history`, `save_digest`, `delete_digest`) |
| **Skill (executa)** | [`executas/research-coach/SKILL.md`](executas/research-coach/SKILL.md) — chat behavior that drives the tool |
| **`tools.invoke`** | UI calls the tool to synthesize and manage digests |
| **`storage`** | Persistent digest library |
| **`chat.write_message`** | App posts a confirmation back to Anna chat on save |
| **Permissions** | `storage.read`, `storage.write`, `tools.invoke`, `chat.write_message` |

│   ├── style.css                    # Dark-theme UI

**Data flow:** UI (`app.js`) → `anna.tools.invoke` → `research-processor` Tool (`index.js`) → Anthropic API → typed JSON digest → rendered as cards → on approval, persisted via the Tool's storage and echoed to Anna chat.

## Run It Locally (fastest demo)

No Anna account required — `run-local.js` serves the UI and proxies tool calls to the Node plugin over stdio, using a local mock of the Anna SDK.

**Prerequisites:** Node.js 18+ and an Anthropic API key.

```bash
# 1. Install the tool's dependencies
cd executas/research-processor-node
npm install
cd ../..

# 2. Set your Anthropic API key
#    Windows PowerShell:
$env:ANTHROPIC_API_KEY = "sk-ant-..."
#    macOS / Linux:
export ANTHROPIC_API_KEY=sk-ant-...

# 3. Start the local demo
node run-local.js
```

Then open **http://localhost:3000** and research any topic.

**Demo path to try:** enter *"Large Language Models"* (Standard) → review the cards → **Save to Library** (it appears in the sidebar + a chat confirmation logs to console) → click a **Related Topic** to chain into the next digest → reopen a saved item from the sidebar.

## Deploy on Anna

1. **Mint executa IDs** at [anna.partners/executa](https://anna.partners) — a Tool and a Skill. *(Already wired in this repo: `tool-kataenda-research-processor-kfj4mr5e`, `skill-kataenda-research-coach-r6tq2m34`.)*
2. **Add your `ANTHROPIC_API_KEY`** to the Tool's credentials in Anna.
3. **Upload the bundle** — create a new app version under *My Apps*, paste [`manifest.json`](manifest.json), and upload everything in [`bundle/`](bundle/) (the `mock-sdk.js` file is harmless but only used locally).
4. **Install & open** the app from your Anna sidebar and start researching.

## Deploy to Production (Coolify / Docker)

The repo also ships a standalone web server ([`server.js`](server.js)) so the app can run as a normal containerized web service — useful for a public live demo. It serves the UI, runs the `research-processor` plugin as a child process, and proxies tool calls over stdio (the Anna SDK is swapped for a local shim at runtime).

**Coolify steps:**

1. **New Resource → Application → Public/Private Repository** → point it at `https://github.com/kataenda/anna-research-digest`.
2. **Build Pack: `Dockerfile`** (the repo includes a [`Dockerfile`](Dockerfile)). Nixpacks also works — `npm install` runs the plugin's deps via `postinstall`, and `npm start` launches the server.
3. **Environment variables:**
   - `ANTHROPIC_API_KEY` = your Claude API key *(required)*
   - `PORT` = `3000` *(optional; Coolify usually sets this automatically)*
4. **Port:** the app listens on `3000` (and binds `0.0.0.0`).
5. **Health check path:** `/health` (returns `{"status":"ok"}`).
6. **Deploy**, then attach your domain. Done.

> **Note on persistence:** saved digests live in the container filesystem (`~/.anna/research-digest/state.json`). For the library to survive redeploys, mount a Coolify **persistent volume** at `/root/.anna`. For a demo this is optional.

Run the same image locally:

```bash
docker build -t research-digest .
docker run -p 3000:3000 -e ANTHROPIC_API_KEY=sk-ant-... research-digest
```

## Tech Notes

- **Model:** `claude-opus-4-8` via the Anthropic SDK. Swap models on the `model:` line in [`executas/research-processor-node/index.js`](executas/research-processor-node/index.js).
- **Tool protocol:** JSON-RPC 2.0 over stdio (`describe`, `research`, `get_history`, `save_digest`, `delete_digest`).
- **Storage:** digests persist in the Tool's local state (`~/.anna/research-digest/state.json` in local mode; Anna storage when deployed).
- **Security:** output is HTML-escaped before rendering; the API key is read from the environment / Anna credential store and never shipped in the bundle.
