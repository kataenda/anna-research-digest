# Research Digest AI

> An AI-native research assistant on **Anna** that turns any topic into a structured, reviewable knowledge digest — summary, key points, key concepts, and related ideas — then lets you approve and save it to a personal library.

Built for the **Anna AI-Native App Hackathon** (Jun 15–22, 2026).

▶️ **[Live demo](https://annaresearch.soenic.com)** · 🎬 **[Demo video](https://youtu.be/QdnF-ZqskO8)**

---

## TL;DR for Judges

- **What it is:** A focused research workflow, not a chatbot. You enter a topic → AI synthesizes a structured digest → **you review and approve** → it's saved to your library with persistent state.
- **Why it's AI-native:** The AI is the engine of every result. It produces **structured output** (typed JSON → UI cards), not free-form chat.
- **How it fits Anna:** It **borrows the Anna AI runtime** — research is generated by the host LLM via `anna.llm.complete`, the library persists via `anna.storage`, and a `SKILL.md` drives in-chat behavior. No custom backend, no API key.
- **Try it in 60 seconds:** Run the standalone build locally — one command.

## The Workflow (one clear interaction)

```
   ┌─────────────┐    ┌──────────────────┐    ┌─────────────────┐    ┌───────────────┐
   │ Enter topic │ →  │  AI synthesizes  │ →  │  Human review   │ →  │ Save / chain  │
   │ + pick depth│    │ structured cards │    │ approve/discard │    │ to library    │
   └─────────────┘    └──────────────────┘    └─────────────────┘    └───────────────┘
```

1. **Enter a topic** — e.g. *"Byzantine fault tolerance"*, *"CRISPR gene editing"*, *"Stoic philosophy"*.
2. **Choose depth** — Quick (3 points), Standard (5), or Deep Dive (7).
3. **AI synthesizes** a digest into four structured cards:
   - **Summary** — 2–3 sentence executive overview
   - **Key Points** — the most important takeaways
   - **Key Concepts** — term + definition glossary
   - **Related Topics** — clickable suggestions to chain research
4. **Human review** — approve to **Save to Library**, or **Discard**. Nothing is stored until you say so.
5. **Library** — browse, reload, or delete saved digests in the sidebar; click a related topic to start the next digest instantly.

## How AI Is Used (meaningfully, not decoratively)

- Every digest is generated by the **Anna host LLM** via `anna.llm.complete` — the app *borrows the Anna AI runtime* instead of shipping its own model or API key.
- The model is constrained to return **strict, typed JSON** (`title`, `summary`, `key_points`, `concepts`, `related_topics`, `confidence`) which the UI renders as structured cards — this is **AI producing structured app state**, not free-form chat.
- A **confidence signal** (`high | medium | low`) is surfaced per digest so the human reviewer can judge trustworthiness.
- The **`research-coach` Skill** lets the same capability work conversationally inside Anna chat: it decides depth and enforces the human-review handoff.

## How It Fits Anna

| Anna primitive | Where it's used |
|---|---|
| **App + Manifest (schema v2)** | `manifest.json` — permissions, views, `host_api` |
| **Host LLM** — `anna.llm.complete` | Generates every digest (structured JSON) |
| **Host storage** — `anna.storage.get/set` | Persistent digest library |
| **Skill (executa)** | `executas/research-coach/SKILL.md` — in-chat depth selection + human-review protocol |
| `anna.chat.write_message` | App posts a confirmation back to Anna chat on save |
| `anna.window.set_title` | Window title reflects the current digest |
| **Permissions** | `llm.complete`, `storage.read`, `storage.write`, `chat.write_message`, `ui.svg` |

> There is **no custom Tool/Executa for the AI** — the app runs entirely on Anna host APIs. This is the "borrow an AI runtime" pattern.

## Architecture

```
anna-research-digest/
├── app.json                 # App metadata (slug, name, category, bundled skill)
├── manifest.json            # Anna AppManifest (schema v2 — permissions, views, host_api)
├── server.js                # Standalone host (prod / Coolify): serves UI + POST /llm → Claude
├── run-local.js             # Local dev entry (thin wrapper around server.js)
├── Dockerfile               # Production container image
├── package.json
├── bundle/                  # The App UI (static SPA — HTML/CSS/JS)
│   ├── index.html           # Layout: search panel, digest cards, library sidebar
│   ├── app.js               # Anna SDK: anna.llm.complete (synthesis) + anna.storage (library)
│   ├── style.css            # Dark-theme UI
│   ├── logo.png             # App / header logo
│   ├── favicon.ico          # Favicon (+ favicon-16/32.png)
│   ├── icon.svg             # App icon
│   └── mock-sdk.js          # Standalone-only SDK shim (llm.complete → /llm, storage → localStorage)
└── executas/
    └── research-coach/
        └── SKILL.md          # The Skill (in-chat behavior)
```

**Data flow (inside Anna):** UI (`app.js`) builds a strict-JSON prompt → `anna.llm.complete` → Anna host LLM → typed JSON digest → rendered as cards → on approval, persisted via `anna.storage` and echoed to Anna chat with `anna.chat.write_message`.

**Data flow (standalone):** the same `app.js` runs unchanged; `mock-sdk.js` maps `anna.llm.complete` → `POST /llm` (which `server.js` answers by calling the Claude API) and `anna.storage` → browser `localStorage`.

## Run It Locally (fastest demo)

The standalone build runs the **exact same UI** as inside Anna; `server.js` stands in for Anna's host LLM by calling Claude.

**Prerequisites:** Node.js 18+ and an Anthropic API key.

```bash
# 1. Set your Anthropic API key (used only by the standalone host)
#    Windows PowerShell:
$env:ANTHROPIC_API_KEY = "sk-ant-..."
#    macOS / Linux:
export ANTHROPIC_API_KEY=sk-ant-...

# 2. Start it (no dependencies to install — server.js uses Node's built-in fetch)
npm start          # or: node run-local.js
```

Then open **http://localhost:3000** and research any topic.

**Demo path to try:** enter *"Large Language Models"* (Standard) → review the cards → **Save to Library** (it appears in the sidebar) → click a **Related Topic** to chain into the next digest → reopen a saved item from the sidebar.

## Deploy on Anna

The app is published on Anna using the `anna-app` CLI:

```bash
anna-app apps push          # upsert the working draft (manifest + bundle)
anna-app apps cut 0.8.0     # snapshot an immutable version
# then Publish the version from the Developer Console → Versions tab
```

Because the app uses the **host LLM** and **host storage**, there is nothing else to configure — no API key, no tool credentials. Install it from your Anna sidebar (or `#research-digest` in chat) and start researching. The `research-coach` Skill is bundled and frozen with each version.

## Deploy to Production (Coolify / Docker)

`server.js` runs the app as a normal containerized web service for a public live demo. It serves the UI and implements `POST /llm` (the stand-in for `anna.llm.complete`) by calling the Claude API.

**Coolify steps:**

1. **New Resource → Application → Public/Private Repository** → point it at `https://github.com/kataenda/anna-research-digest`.
2. **Build Pack: Dockerfile** (included). Nixpacks also works (`npm start` launches `server.js`).
3. **Environment variables:**
   - `ANTHROPIC_API_KEY` = your Claude API key *(required for the standalone host)*
   - `ANNA_LOCAL_MODEL` = Claude model id *(optional, default `claude-opus-4-8`)*
   - `PORT` = `3000` *(optional; Coolify usually sets this)*
4. **Port:** `3000` (binds `0.0.0.0`). **Health check path:** `/health` (returns `{"status":"ok"}`).
5. Deploy, then attach your domain.

> Standalone persistence is per-browser (`localStorage`), so no server-side volume is needed for the demo.

Run the same image locally:

```bash
docker build -t research-digest .
docker run -p 3000:3000 -e ANTHROPIC_API_KEY=sk-ant-... research-digest
```

## Tech Notes

- **Inside Anna:** synthesis uses the session's host LLM via `anna.llm.complete` (no API key in the app); the library uses `anna.storage`.
- **Standalone:** `server.js` calls the Claude API with `ANTHROPIC_API_KEY` and returns the same MCP-shaped result (`{ content: { type: 'text', text } }`); the model is set via `ANNA_LOCAL_MODEL` (default `claude-opus-4-8`).
- **Structured output:** the model is instructed to return JSON only; the UI parses it and renders cards (with HTML-escaping before render).
- **No backend dependencies:** `server.js` uses Node's built-in `fetch` — `npm install` pulls nothing.
